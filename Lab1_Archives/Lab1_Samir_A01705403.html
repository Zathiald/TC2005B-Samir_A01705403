Respuestas a preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de dispositivos interconectados que permite la comunicación y el intercambio de información. Por otro lado, la World Wide Web (WWW) es un servicio que se ejecuta en Internet y permite a los usuarios acceder y compartir información a través de páginas web utilizando navegadores.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Este método solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
  • HEAD: El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
  • POST: Este método se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
  • PUT: El método PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
  • PATCH: El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.
  • DELETE: El método DELETE borra un recurso en específico.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código de respuesta HTTP 200 significa que la solicitud fue procesada con éxito por el servidor. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un error 404. Este error indica que el recurso solicitado no se encontró en el servidor, lo cual puede ser el resultado de un enlace roto o de un recurso que fue eliminado. El desarrollador debe asegurarse de que todos los enlaces estén actualizados y funcionen correctamente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, también es responsabilidad del desarrollador corregir un error 500. Este error indica un problema en el servidor, y el desarrollador debe investigar y solucionar el problema.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo HTML5 depreciado significa que ya no se recomienda su uso y puede ser eliminado en futuras versiones de HTML. Algunos elementos de HTML 4 que están desaprobados en HTML5 incluyen frame, frameset, y font.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 introdujo una serie de nuevas etiquetas semánticas (como header, footer, article, y section), soporte para gráficos y multimedia (con canvas y video), y APIs para JavaScript (como geolocalización y almacenamiento local).

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla en HTML se compone de varias partes, incluyendo table para crear la tabla, tr para las filas, td para las celdas de datos, y th para las celdas de encabezado. También puedes usar thead, tbody, y tfoot para agrupar partes de la tabla.

¿Cuáles son los principales controles de una forma HTML5?

HTML5 introdujo varios nuevos tipos de controles para los formularios, incluyendo email, tel, number, url, range, date, time, color, y otros.

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información generalmente incluye las siguientes etapas: análisis de requisitos, diseño, implementación, pruebas, despliegue, y mantenimiento.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información es similar al ciclo de vida, pero se centra más en el desarrollo del sistema. Incluye las etapas de planificación, análisis, diseño, desarrollo, pruebas, implementación, y mantenimiento.
